ui.Tooltip

Use tooltips to display information messages anywhere in the UI.

Installation

Include joint.ui.tooltip.css and joint.ui.tooltip.js files to your HTML:


<link rel="stylesheet" type="text/css" href="joint.ui.tooltip.css" />
<script src="joint.ui.tooltip.js"></script>

Creating a Tooltip

A tooltip can be created with the joint.ui.Tooltip constructor. See below for options that you can pass to the constructor.


new joint.ui.Tooltip({
    target: '.top-tooltip',
    content: 'Top directed tooltip.',
    top: '.top-tooltip',
    direction: 'top'
});

The joint.ui.Tooltip constructor takes the following parameters:

  • target - A CSS selector or DOM element or jQuery object that is the target for the tooltip. When the user hovers over this element with his mouse cursor, the tooltip appears.
  • rootTarget - A CSS selector or DOM element or jQuery object that is the root target for the tooltip. When using the rootTarget, the event handlers that trigger the tooltip display will be delegated (as opposed to directly-bound). This is useful when the target is a selector pointing to elements that might get removed from the DOM and then put back again. The tooltip tries to find the selected elements specified by the selector in the target option when it is initialized (during instantiation) and than uses these element as the target for the tooltip. If later on such elements are removed from the DOM the tooltip looses a reference to the target. Therefore, we can prevent this by using rootTarget for selecting the container element and target selector for selecting the real target of the tooltip at the time of the "mouseover" event.
  • content - The content of the tooltip. It can be an arbitrary HTML.
  • padding - The distance between target element and the tooltip.
  • direction - The direction of the tooltip arrow. Can be either top, right, bottom or left.
  • left - A CSS selector or DOM element or jQuery object or a function used to compute the position of the left edge of the tooltip. In other words, the left selected element is on the left of the tooltip. It's usually the same as target. If the left is a function, it is passed the real target and is expected to return an element which will be positined on the left of the tooltip. Usually, you use a function if the target selects more that one element in which case you'd like the final tooltip to appear on the right of the current target which you don't know before. This is extremelly useful if you, for example, want the tooltip to appear after some other element.
  • top, right, bottom - Analogous to left

Tooltips in Halo

It is very useful to show the user a hint on the Halo icon tools. Fortunately, it is very easy to do: 


// ... where you create your halo ....

var halo = new joint.ui.Halo({ graph: graph, paper: paper, cellView: cellView });

new joint.ui.Tooltip({
    className: 'tooltip small',
    target: halo.$('.remove'),
    content: 'Click to remove the object',
    direction: 'right',
    right: halo.$('.remove'),
    padding: 15
});

Notice we use the 'tooltip small' className in our example. JointJS tooltips come pre-built with two sizes of tooltips: small and normal. By using the 'tooltip small' className, we enabled the small version.
